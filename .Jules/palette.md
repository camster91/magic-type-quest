## 2025-05-22 - Global Keyboard Navigation & Status Accessibility
**Learning:** For immersive, full-screen applications, users (especially keyboard-driven ones) expect a consistent "exit path" like the Escape key to return to a safe state (Main Menu). Additionally, purely visual status indicators like health hearts and achievement toasts are invisible to screen readers unless wrapped in a container with a dynamic `aria-label` or `role="status"`.
**Action:** Always implement a global `Escape` key listener for screen navigation and ensure visual-only indicators have descriptive, live ARIA counterparts. Use `aria-hidden="true"` on decorative icon sub-elements to prevent redundant announcements.
